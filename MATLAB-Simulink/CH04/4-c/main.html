Topic 1 : 직렬 연결된 시스템의 샘플 데이터 모델

이번 파일에서는 다음의 1차 시스템 2개를 직렬로 연결한 연속 시간 시스템의 샘플 데이터 모델에 대해 이해해보고자 합니다.
,
Gp1_tf = tf(1,[1 1]);
Gp2_tf = tf(1,[1 2]);
두 시스템을 직렬로 연결한 연속 시간 시스템의 전달함수는 * 연산자를 이용하여 손쉽게 얻을 수 있습니다.
Gp_series_tf = Gp1_tf * Gp2_tf
Gp_series_tf = 1 ------------- s^2 + 3 s + 2 연속시간 전달 함수입니다. 모델 속성
이 시스템의 펄스 전달함수는 c2d 함수를 이용하여 아래와 같이 계산됩니다.
T = 0.2;
Gd_series_tf = c2d(Gp_series_tf, T, 'zoh')
Gd_series_tf = 0.01643 z + 0.01345 ---------------------- z^2 - 1.489 z + 0.5488 샘플 시간: 0.2 seconds 이산시간 전달 함수입니다. 모델 속성
여기서 주목할 점은, 위의 펄스 전달함수 는 와 의 펄스 전달함수를 각각 계산하고 이를 곱한 아래의 전달함수와 다르다는 것입니다. (그 이유에 대해 잘 고민해봅시다.)
c2d(Gp1_tf, T, 'zoh') * c2d(Gp2_tf, T, 'zoh')
ans = 0.02988 ---------------------- z^2 - 1.489 z + 0.5488 샘플 시간: 0.2 seconds 이산시간 전달 함수입니다. 모델 속성
한편 ss 모델 객체를 이용하여 두 1차 시스템의 상태변수 모델을 계산하고, 이를 동일한 방법으로 직렬 연결하여 최종 상태변수 모델을 얻어봅시다.
Gp1_ss = ss(Gp1_tf);
Gp2_ss = ss(Gp2_tf);
 
Gp_series1_ss = Gp1_ss * Gp2_ss
Gp_series1_ss = A = x1 x2 x1 -1 1 x2 0 -2 B = u1 x1 0 x2 1 C = x1 x2 y1 1 0 D = u1 y1 0 연속시간 상태공간 모델입니다. 모델 속성
또 하나의 흥미로운 점은, 곱하는 순서를 반대로 하였을 때에는 의 값들이 달라진다는 점입니다. (이 부분도 이유에 대해 고민해봅시다.)
Gp_series2_ss = Gp2_ss * Gp1_ss
Gp_series2_ss = A = x1 x2 x1 -2 1 x2 0 -1 B = u1 x1 0 x2 1 C = x1 x2 y1 1 0 D = u1 y1 0 연속시간 상태공간 모델입니다. 모델 속성
하지만 이들의 전달함수를 계산해보면 그 둘은 동일함을 알 수 있습니다.
(즉, Gd_series1_ss와 Gd_series2_ss는 동일한 시스템에 대한 서로 다른 상태변수 모델이라고 이해할 수 있습니다.)
tf(Gp_series1_ss)
ans = 1 ------------- s^2 + 3 s + 2 연속시간 전달 함수입니다. 모델 속성
tf(Gp_series2_ss)
ans = 1 ------------- s^2 + 3 s + 2 연속시간 전달 함수입니다. 모델 속성
이제 위의 연속 시간 시스템과 0차 홀드, 샘플러가 합쳐진 샘플 데이터 시스템의 상태변수 모델을 아래와 같이 얻읍시다.
Gd_series1_ss = c2d(Gp_series1_ss, T, 'zoh')
Gd_series1_ss = A = x1 x2 x1 0.8187 0.1484 x2 0 0.6703 B = u1 x1 0.01643 x2 0.1648 C = x1 x2 y1 1 0 D = u1 y1 0 샘플 시간: 0.2 seconds 이산시간 상태공간 모델입니다. 모델 속성
본 파일에서는 별도로 다루지 않지만, 두 시스템을 병렬로 연결하여 얻은 연속 시간 시스템 역시 유사한 방식으로 해석할 수 있습니다.
(이 경우 두 모델 객체에 + 연산자를 사용하여 병렬로 연결시킵니다.)
